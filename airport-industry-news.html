RULE — All future GTM-GA4 work: data interruptions must be flagged and confirmed before any change is made Any GTM change that causes a registered GA4 custom dimension to stop receiving data is a data interruption. Before making such a change: (1) identify which registered dimensions are affected and state the exact date data will stop; (2) confirm whether any Looker Studio report uses that dimension; (3) get explicit confirmation to proceed. This rule was established during Railway-News v204 work and applies to all containers.
!
1-year parameter transition rule Any GA4 parameter rename requires dual-parameter sending for a minimum of 1 year before the old parameter is removed. Client contracts are 1 year. Looker Studio reporting must cover the full contract period for any active client. This applies to any future standardisation of ain_client_name / client_logo_view → client.

1
Apollo.io Tag (47) has no consent requirement — fires on all pages Tag 47 is the Apollo.io B2B visitor identification snippet (App ID: 66336ab7c350390572a05d4b). It fires on All Pages with consentStatus: NOT_SET. Apollo.io is a US-based tool — their GTM snippet ships without GDPR consent controls as standard. Under GDPR and UK GDPR, processing EU visitor data for B2B identification requires either consent or a documented Legitimate Interests Assessment. As data controller, the consent gate must be added by us.

Fix: Add ad_storage consent requirement to Tag 47. This is the same fix applied to Railway-News GTM-KQ4VZD5 Tag 195. Requires confirming that the airportindustry-news.com CMP signals ad_storage. If not, use analytics_storage as a pragmatic fallback.
2
Client Pages trigger (64) — broken /suppliers/ exclusion Trigger 64 is a PAGEVIEW trigger that fires on supplier profile pages (firing client_page_view, Tag 66). It excludes /suppliers/ using EQUALS /suppliers/. This only excludes the exact path /suppliers/ (the supplier listing page). Every supplier detail page at /suppliers/company-name/ is not excluded — these pages incorrectly fire client_page_view as well.

Fix: Change the exclusion condition from EQUALS /suppliers/ to CONTAINS /suppliers/. This is the same bug found and fixed in Railway-News GTM-KQ4VZD5 Trigger 225 (v204).
1
All 13 GA4 event tags have consentStatus: NOT_SET Only the config tag (Tag 68) requires analytics_storage. Event tags with NOT_SET are treated as "not requiring consent" and fire regardless of visitor choice — completely bypassing the consent gate on the config tag. All 13 event tags need analytics_storage added. Note: the config tag is already correctly configured, so this is a straightforward bulk update to the event tags only.
2
All 13 GA4 event tags hardcode measurementIdOverride: G-GQEFW14X43 If the GA4 property ever changes, all 13 tags require manual editing. Best practice: remove measurementIdOverride from all event tags and let them inherit the Measurement ID from the config tag (Tag 68). Single update point. Same fix was applied in Railway-News v204.
