Next Steps

initial audit complete · 2 critical fixes needed · gaps to assess ▼
Immediate — Required Before Next Publish
1
Add analytics_storage consent requirement to all 10 GA4 event tags All GA4 event tags (IDs 6, 11, 16, 20, 24, 27, 29, 31, 36, 40) have consentStatus: NOT_SET. This means they fire on every page regardless of visitor consent — completely bypassing the consent gate on the config tag (Tag 41). Under GDPR, analytics tracking must require consent. This must be fixed before any new version is published.
2
Fix Trigger 9 (Client Pages) — change EQUALS /suppliers/ to CONTAINS /suppliers/ The Client Pages trigger excludes /suppliers/ using an EQUALS condition. This only excludes the exact path /suppliers/ — all supplier detail pages at /suppliers/company-name/ are not excluded, so client_page_view fires incorrectly on those pages. The same bug existed in the Railway-News container and was fixed in v204.
High Priority
!
Remove measurementIdOverride from all 10 GA4 event tags All event tags hardcode measurementIdOverride: G-CMEZMGPMVC. If the GA4 property ever changes, all 10 tags need individual edits. Best practice: remove the override from event tags and inherit the Measurement ID from the config tag (Tag 41). One change point rather than ten.
!
Rename min_client_name parameter on Tag 36 to client The supplier_content_impression event (Tag 36) sends the client name as min_client_name. The client_page_view event (Tag 11) uses client for the same concept. All other containers in the network use client. Before renaming: check whether a GA4 custom dimension is already registered for min_client_name. If it is and has active Looker Studio reports, apply the 1-year dual-parameter transition (send both min_client_name and client simultaneously for at least 1 year before dropping the old name).
Assess — Tracking Gaps vs Network
GAP
Form tracking — CF7 form submissions not tracked Railway-News uses a Custom HTML tag to push cf7submission to the dataLayer on Contact Form 7 submissions, and a GA4 event tag to capture a form_submission event with form_id. This container has no equivalent. If maritimeindustry-news.com uses CF7 for enquiry forms, this is a high-value gap — form submissions are the clearest signal of commercial intent. Confirm whether CF7 is used before building.
GAP
Banner tracking — AdRotate impressions and clicks not tracked No banner impression or click events exist in this container. If the site uses AdRotate for banner advertising (as Railway-News does), this is a significant commercial reporting gap — clients cannot see impression and click data for their banner placements. Confirm whether AdRotate is used, and if so, implement the same IntersectionObserver-based approach developed for Railway-News v204.
GAP
Magazine tracking — no magazine engagement events No magazine or magazine_pdfs_individual events exist. If the site has a magazine section with flipbook or PDF downloads, that engagement is untracked. Confirm whether a magazine section exists before building.
Low Priority / Housekeeping
UI
Remove console.warn from Variables 5 and 8 The Post Published Date and Author variables both contain console.warn('Error getting ...') in their catch blocks. This produces console noise in production on any page where the expected DOM elements are absent (i.e. all non-article pages). Convert to silent returns.
UI
Standardise path exclusions between Triggers 4 and 9 Trigger 4 excludes both /events-exhibitions/ and /events-exhibition/. Trigger 9 only excludes /events-exhibition (without trailing slash). Confirm which URL pattern the site uses and make both triggers consistent.
Planned — Consent Re-trigger Fix
v+
Apply consent re-trigger fix — same pattern as Railway-News v205 First-session page views are lost when a new visitor accepts consent on their first visit. Railway-News v205 fixes this by adding a {{DLV - analytics}} Data Layer Variable + Consent Granted - Analytics trigger (Custom Event a2b_consent_updated with condition analytics = granted) applied to the GA4 config tag and all page-level / impression tags.

Apply this same pattern to Maritime Industry-News once Railway-News v205 has been confirmed working in GTM Preview. Do not add this trigger to form submission, link click, or download click tags — those should only fire on actual user actions, not on consent acceptance.

★
RULE — All future GTM-GA4 work: data interruptions must be flagged and confirmed before any change is made Any GTM change that causes a registered GA4 custom dimension to stop receiving data is a data interruption. Before making such a change: (1) identify which registered dimensions are affected and state the exact date data will stop; (2) confirm whether any Looker Studio report uses that dimension; (3) get explicit confirmation to proceed. This rule was established during the Railway-News v204 audit (GTM-KQ4VZD5, 20 May 2026).
!
RULE — 1-year dual-parameter transition for any parameter rename Client contracts run for 1 year. Any parameter rename must involve dual-parameter sending for a minimum of 1 year before the old name is removed. If min_client_name has a registered GA4 custom dimension and active Looker Studio reports, renaming to client requires sending both parameters simultaneously from the change date, with the old name removable no earlier than 1 year later.

1
All 10 GA4 event tags have consentStatus: NOT_SET Tags 6, 11, 16, 20, 24, 27, 29, 31, 36, and 40 all have consent NOT_SET. This means they are classified as "not requiring consent" and fire on every page, bypassing the consent gate configured on the config tag (Tag 41). The config tag correctly requires analytics_storage — but if the event tags fire regardless, the consent architecture is ineffective. Under GDPR, analytics tracking that profiles user behaviour requires consent. Action: set analytics_storage as required on all 10 event tags.
2
Trigger 9 (Client Pages) uses EQUALS /suppliers/ — supplier detail pages fire client_page_view incorrectly The exclusion condition {{Page Path}} EQUALS /suppliers/ only excludes the exact path /suppliers/. All supplier detail pages at /suppliers/company-name/ pass through the exclusion and incorrectly fire client_page_view. This inflates client page view counts with visits to the suppliers directory. Action: change the exclusion condition to CONTAINS /suppliers/.
1
All 10 GA4 event tags hardcode measurementIdOverride: G-CMEZMGPMVC If the GA4 property ever changes, all 10 tags must be individually updated. The correct pattern: remove measurementIdOverride from event tags and inherit from the config tag (Tag 41). This reduces the risk of partial updates leaving some tags pointing at the wrong property.
2
min_client_name parameter on Tag 36 is non-standard The supplier_content_impression event sends the client name as min_client_name. Every other event on this site and across the network uses client for the same concept. Before renaming: confirm whether a GA4 custom dimension is registered for min_client_name. If it has active Looker Studio reports, apply the 1-year dual-parameter transition rule — send both min_client_name and client simultaneously for at least 1 year before dropping the old name.
1
Variables 5 (Post Published Date) and 8 (Author) contain console.warn(...) in their catch blocks. This fires on every non-article page where the expected DOM elements don't exist, producing console noise in production. Remove or convert to silent return undefined.
2
Trigger 4 (Post Pages Only) excludes both /events-exhibitions/ and /events-exhibition/. Trigger 9 (Client Pages) only excludes /events-exhibition (without trailing slash). Confirm which URL pattern the site uses and make both triggers consistent.
3
Trigger 18 (HP Featured Suppliers) uses a 5% visibility threshold vs 10% used by all other visibility triggers in the container. Very low threshold — a banner barely entering the viewport edge may trigger. Consider aligning to 10% for consistency.
